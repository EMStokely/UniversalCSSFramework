Universal CSS Framework

Last Updated: 7/7/2022

Author: Mitchell Stokely

Summary: This is a project I call the Universal Cascading Style Sheet Framework. It is simply a small package of CSS files you drag-and-drop into your website, and which "resets" the web browser's HTML element designs so they consistently display your website's elements in many types of web browsers, past and present. This CSS system also offers you a more advanced CSS system for managing how older and newer browsers view your website's HTML and CSS by hiding modern CSS designs from older browsers and degrading gracefully your HTML layouts for those user agents. Using its carefully coded CSS import system, this framework provides superior cross-browser style sheet display and support of many types of web designs in browsers going back to the birth of the World Wide Web (mid-1990's) all the way up to modern HTML5 ones built today. As such, a web developer using this system will not have to struggle to modify CSS and HTML to fit the needs of troublesome, older browsers from the past that failed to render layouts correctly. If you are spending thousands of hours designing your websites to look good in all the old, antiquated browsers then give my CSS system a try. It will simplify your coding life by removing the need to do so in most cases, and allow you to focus more of your time doing cutting-edge CSS web page layout and design for the newest HTML5 browsers.

I realize many younger developers laugh at the thought of designing for even old Internet Explorer (v. 9-11), which was fully deprecated by Microsoft as a product only recently. Why play russian roulette with your website in those situations and watch your carefully constructed designs be mangled live in front of an audience of very important people? Instead of packing gigantic clunky JavaScript API's into your projects to fix this type of problem and hope scripts will rewrite your CSS for you or design websites for the newest most popular Chrome browsers while ignoring the rest, what if you could deliver readable text and media content to over 99% of all web viewers and screenreaders known around the world?

The fact is these cross-browser battles are never truly over, even today. I recently went to a major cryptocurrency product site that crashed and burned with a nasty JavaScript and layout error in three different web browsers on my phone! JavaScript does not have all the answers! The sad fact is, cross-browser issues will be with us forever, or at least as long as there are untrained developers still coding cutting-edge tech into websites that is not correctly tested and fixed in multiple web browsers. Many modern browsers still do not support big chunks of HTML5, ECMAScript, and CSS3! In the case of CSS, we must stop wasting time testing or re-coding layouts for older browsers that are over 10 years old. Yet, many will be around for many more years to come. You risk diminishing your audience and your content by failing to design for antiquated browsers. You risk showing up at an event, opening a mobile device, or logging into a PC with an older version of a browser that crashes your fancy layout in front of hundreds of customers. That is what my CSS framework tries to prevent, and why I hope you find my solution helpful in providing some relief from dealing with old browsers but with minimal work. Warning: This solution does not fix fancy CSS layouts for you, redesign grids or flex layouts in IE, or help you struggle to fix JavaScript API that fails in some smart phone browser. It simply hides cutting edge CSS from older agents while delivering clean white text pages that are clear and readable to those users, while resetting all the past and modern HTML elements so they look the same across dozens of browsers. This allows you to forget how your content will look in most of the troublesome older browsers and viewers that might pop up. You can then give 100% focus on building rich layouts for the modern browsers you care about now. Forget customizing layouts for the older browsers! That is what my new CSS system will allow you to do. But my message remains.....YES, you should care about antiquated browsers! And your layouts should "degrade gracefully" in older browsers so a wider range of users can access the valuable content in your site rather than struggling with elaborate scripts that hide valuable text content away from viewers because you wanted them to see your fancy designs. If they cannot or do not process CSS or HTML correctly, they will fail. Why should you punish them or exclude older browsers from seeing your content? Delivering clean content for everyone will benefit your web project and best of all, allow you to brag and say your website is readable in 99.99% of web browsers!

As such, this style sheet system is a complete and very comprehensive CSS framework designed for cross-browser style sheet support in browsers going back to the birth of the World Wide Web (since the 1990's). It provides old browsers the minimum styles needed to display HTML and content correctly, while adding newer styles for more modern browsers that can display more modern CSS features and layouts. I call this the "Universal CSS Framework" because it was designed to universally support as many web browsers as possible going back over time, including browsers that did not support CSS in the mid-1990's when the first web browsers appeared on the scene. This system should provide some form of basic layout or design in all browsers, old and new, with a rich cascading style sheet system that degrades gracefully to simpler layouts for older web browsers, while providing rich support for cutting-edge CSS3 features in newer ones. As such, this CSS system is a very simple solution to a complex problem because it simple delivers plain white content pages to older browsers while fully enabling the newer HTML5 browsers the power to move forward with complex layout and design.

The way this new Cascading Style Sheet system works is it first delivers a very clean, thoroughly tested set of basic "reset" styles coded in very simple CSS1 and CSS2. These styles reset basic HTML elements styles that all web browsers and user agents use going back to the mid-1990's and early 2000's up till today. These reset HTML element designs are read by modern HTML5 browsers as well, resetting their element designs to match what older agents support. Using "@import", newer browsers are then provided an additional, more modern "reset" set of sheets, as well as the ability to process more advanced CSS that is hidden from older browsers using CSS "@import" filtering code. This layered "reset" CSS framework simplifies basic HTML elements to core style designs that reduce all layouts to plain white content pages all the older browsers recognize (this includes Internet Explorer 1-7, pre-HTML5 browsers before 2010, IE 5 series for Mac , Netscape 4 series, etc.). Text and media for older browsers then appears as vertically stacked content in simplified web layouts that allow users to access your content. But the import filters hide newer sheets which allow more modern HTML5 browsers access to more advanced CSS3 features and HTML layouts. The system then allows you to paste in all your cutting edge CSS layouts and custom CSS for new HTML5 agents, mobile browsers, etc. Again, it does this using a handful of "@import" rules that are not readable by a wide range of older browsers. All of the older browsers read your plain HTML ok, but are prevented from accessing newer CSS sheets that would display your web pages in more complex layout designs. In many cases older browsers fail to render such newer design correctly and jumble your text content, or use proprietary JavaScript or HTML code that is not understood or rendered correctly. Instead, this CSS framework delivers your page content as plain white, stacked, text-based content blocks, while modern browsers and HTML5 agents and devices will see your rich layouts exactly as designed. My system when combined with my HTML tutorial template code (see below) allows you to also deliver content to screenreaders reliably that do not read or render CSS (for the blind). And, it includes mobile design styling, and has a built in print sheet as well. This CSS framework, plus the HTML template combined, can together provide content delivery to 99.99% of all browsers, user-agents, and screen readers, allowing your web content to reach a huge audience with nearly zero customization effort needed in addressing out-dated and deprecated browsers technology.

Using this style sheet framework, older browsers will get plain white, vertical content pages with san-serif black fonts, while newer and more modern browsers will get your cutting-edge, CSS3 designs. This system achieves that using special CSS rules that older browsers do not understand. These newer style sheets are hidden from the old browsers using a special "@import" style sheet trick all known modern browsers understand but older ones do not. The older browsers get a simpler reset element style sheet, while modern browsers get a more advanced reset element sheet, plus your custom CSS layout styles. This includes full support for advanced mobile and print designs using CSS. The following older browsers are now hidden from these newer style sheets using @import rules in this CSS framework:

You should always build websites to work in older browsers starting with a simple baseline design that works in all but delivers advanced features to newer user agents, while allowing CSS layouts to get simpler in older browsers. Most developers have been doing this for over 20 years. So ignore these terms. They are the same thing as the older term, "cross-browser compatibility". But if you insist on asking if this solution does those two things, then the answer is YES!
